# Data Sources

> **This is a portfolio demonstration using public or simulated data. It is not an official emergency management product and should not be used for operational decision-making.**

## What this project uses today

All bundled data is **simulated** and generated by
`scripts/01_create_project_structure.py` with a fixed random seed. Town centroids
are approximate public coordinates; facility/shelter/hospital names and locations
are illustrative; incidents are randomly generated and flagged `simulated_flag =
Y`; hazard zones are hand-drawn rectangles. Nothing sensitive is used.

This keeps the project safe to publish while still exercising a realistic
end-to-end workflow.

## How to swap in real, authoritative public data

The pipeline is built so you can replace the demo with real data **without
changing code**: drop a file with the same name and columns into `data/raw/`, and
`scripts/02_load_clean_data.py` will prefer it over the simulated copy. Below are
safe, public, well-known sources appropriate for a New Hampshire public-safety
project.

| Layer | Recommended public source | Notes |
|---|---|---|
| Town / municipal boundaries | **NH GRANIT** (NH statewide GIS clearinghouse, granit.unh.edu) | Authoritative NH boundaries |
| Roads / corridor | **NH GRANIT** roads, or **OpenStreetMap** (Geofabrik extracts) | OSM is open-licensed |
| Hospitals & EMS | **HIFLD Open** (Homeland Infrastructure Foundation-Level Data) | Public critical-infrastructure layers |
| Fire / police stations | **HIFLD Open**, or local open-data portals | Verify currency |
| Flood hazard zones | **FEMA National Flood Hazard Layer (NFHL)** | Replaces the simulated floodplains |
| Severe weather | **NWS / NOAA** alert polygons (api.weather.gov) | Live alert geometries |
| Demographics | **US Census / ACS** (TIGER/Line + tables) | Population, vulnerability indicators |
| Shelters | **American Red Cross / state EM open data** where published | Confirm it is public & non-sensitive |

### Mapping source fields to this schema

When you download a real dataset, map its columns to the names in
`docs/data_dictionary.md` (e.g. rename `FACILITY` → `name`, `LAT`/`LON` →
`latitude`/`longitude`). Keep `data_source` and `last_updated` populated for
**data lineage**. The cleaning script standardizes casing/whitespace, so minor
formatting differences are handled automatically.

## Data-handling and safety guidance

- **Only use public, non-sensitive data.** Do not ingest 911 CAD data, private
  resident records, or security-sensitive facility details.
- **Respect licenses.** OSM requires attribution (ODbL); HIFLD/FEMA/Census are
  generally public domain — check each layer's terms.
- **Record provenance.** Every record carries `data_source` and `last_updated`
  so a reviewer can trace where a value came from.
- **Re-validate after swapping data.** Real data has real defects; run
  `scripts/03_validate_gis_data.py` and review the QA report before trusting it.
- **Don't overstate accuracy.** Document approximations (e.g. straight-line vs.
  drive-time distance) wherever they exist.

## Reproducing or changing the simulated data

- Change the random seed or validation bounds in `config/config.yml`.
- Edit the `TOWNS` table or the type/severity lists in
  `scripts/01_create_project_structure.py` to re-target another region.
- Re-run `scripts/01` … `09` (or `python scripts/run_pipeline.py`) to regenerate everything downstream.

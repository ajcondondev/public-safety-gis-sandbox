# Changelog

All notable changes to this project are documented here. This is a portfolio
project, so versions mark meaningful milestones rather than software releases.

The format is loosely based on [Keep a Changelog](https://keepachangelog.com/).

## [0.3.0] — Educational tooling & presentation

### Added
- **Outputs portal** (`outputs/index.html`) — a landing page linking the map,
  dashboard, reports, and metadata (generated by script 08).
- **Learning guide** (`docs/learning_guide.md`) — GIS concepts mapped to the
  code, stage-by-stage walkthrough, four-phases-of-emergency-management table,
  and interview prompts.
- **Glossary** (`docs/glossary.md`) and **documentation index** (`docs/README.md`).
- **Architecture diagrams** (`docs/architecture.md`) — Mermaid pipeline, data
  flow, and relational ERD.
- **Optional scripts** (`scripts/optional/`): live NWS public-data ETL
  (`fetch_nws_alerts.py`) and an ArcPy mapbook automation example.
- **CONTRIBUTING.md** and a Mermaid data-flow diagram in the README.

### Changed
- README: educational-guide section, NH context, live-demo (GitHub Pages) notes,
  consistency pass (script ranges, table of contents).

## [0.2.0] — Visuals, tests, CI

### Added
- **Interactive Leaflet web map** and static PNG maps (script 07).
- **Self-contained HTML operational dashboard** (script 08).
- **FGDC/ISO-style metadata** generation (script 09).
- **Automated test suite** (`tests/test_pipeline.py`, 12 tests) and one-command
  **orchestration** (`scripts/run_pipeline.py`, `run.ps1`, `run.sh`, `Makefile`).
- **GitHub Actions CI**, **MIT license**, badges, and `.gitattributes`.

### Fixed
- Analysis layer now excludes out-of-range coordinates (config bounds) so
  validation-failing records can't pollute derived layers or maps.

## [0.1.0] — Initial pipeline

### Added
- Core pipeline (scripts 01–06): generate simulated data → clean → validate
  (QA/QC) → spatial analysis (proximity, hazard overlay, incident priority) →
  dashboard tables + SQLite → summary report.
- SQL schema and QA/summary queries; nine documentation files; exploratory
  notebook; simulated datasets for the Concord–Manchester corridor.
- Safety-first design: all data simulated and labeled; disclaimer throughout.
